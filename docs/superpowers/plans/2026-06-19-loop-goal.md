# Loop / Goal Orchestrator & Worktree Isolation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement persistent goal-driven loops (`/loop`, `/goal`), CLI session commands (`/model`, `/agent`), git worktree isolation, custom agent profiles in `rune.toml`, and Notes HTTP/SSE endpoints.

**Architecture:** A unified `LoopEngine` manages execution states, logs audit records, checks out isolated git worktrees, and coordinates Implementer and Verifier sub-agent sessions using a modular `LoopModeAdapter` interface across CLI and web server platforms.

**Tech Stack:** Rust, Axum, Tokio, Git.

*   Code must format correctly using `cargo fmt --all`.
*   All new modules must reside under `src/loop_engine/` and stay aligned with Rune's zero-trust sandboxed architecture.

    ```rust
    pub struct WorktreeManager {
        pub loop_id: String,
        pub path: std::path::PathBuf,
    }

    impl WorktreeManager {
        pub fn create(loop_id: &str) -> std::io::Result<Self> {
            let path = std::env::current_dir()?.join(".git").join("rune-worktrees").join(loop_id);
            let branch = format!("rune/loop-{}", loop_id);
            // git branch rune/loop-ID
            let _ = std::process::Command::new("git").args(&["branch", &branch]).output();
            // git worktree add path branch
            std::process::Command::new("git")
                .args(&["worktree", "add", path.to_str().unwrap(), &branch])
                .output()?;
            Ok(Self { loop_id: loop_id.to_string(), path })
        }

        pub fn remove(&self) -> std::io::Result<()> {
            std::process::Command::new("git")
                .args(&["worktree", "remove", "--force", self.path.to_str().unwrap()])
                .output()?;
            Ok(())
        }
    }
    ```
